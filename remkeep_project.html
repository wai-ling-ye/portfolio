You use your cloud for storage. What more can cloud storage be? I envision Remkeep as your personal hub to bring together your ideas and experiences.

Objective

Cloud storage can do more than keep your files safe. Remkeep makes it easy to save and organize files and web content in one place using a visual organization system. The ease and flexibility of Remkeep allows your cloud storage to become your go-to dashboard for work and play.

The Problem

Despite the existence of popular, big-name cloud storage services, there are two major problems:

  • Existing cloud storage services allow you to exclusively upload files from your computer (ex. Dropbox) or save links from the web (ex. Pinterest).

  • Cloud storage tends to be messy and cluttered.

Remkeep strives to solve both these problems by allowing you to save any kind of content in a clean, visually organized space.

The Solution

The cloud storage options that exist today act as a simple repository. You put your files into a safe deposit box, and you can view or take those files out of the box later.


But this is not enough.


I want our users to not only have a portable repository, but to use cloud storage service as the central hub for all digital activities. If your personal devices are used to create your own ideas, and the web is where you find ideas and inspiration from others, then our cloud will be the place to bring it all together in a clean, clear, organized space.

“We're really excited at the possibilities we see in the cloud storage and organization market. … [T]here's room for another player if we can find the right combination of features to meet the needs of a particular audience.”

Wireframes

Wireframes provide a simple, visual outline of the layout and navigation of the website. I touch on visual design here, but the main focus is making sure that we deliver information to the user in a clear and intuitive fashion.
As this is the first visual that we will have of the website, I start with sketches, which allows me to generate a large number of layouts from which I can select the ideal layouts for the intended content.

User Testing

User Testing - Round One

In this first round of user testing, I am primarily testing: 1. the clarity of the information architecture and 2. the ease of navigation. To do this, I ask users to perform a series of tasks, which bring them through the various pages of the site. By observing how users navigate the website, I can identify areas that they navigate easily, and areas where they struggle to understand information or navigation.

View the User Testing Script

I tested with 3 users, each of whom provided me with 12-15 issues to be considered for further improvement. I identified the top 3 issues for each user to prioritize for improvement, which included:

Problem

No prices on the upgrade page

Solution

Add prices to the upgrade page

Problem

Landing page is not informative enough about the app

Solution

Have more information or images or videos to explain the features of the app

Problem

Initially unsure of what the upload icon represented

Solution

Improve the iconography of the upload button

Despite the issues encountered, all three users found the website to be intuitive and easy to navigate, and, for the most part, had no problem with performing the tasks. Because of this, I decided to move ahead with the next stage of the design process with the plan to implement the suggested improvements later in the high fidelity prototype.

Brand Strategy

Before I can create the high fidelity prototype, I needed to first finalize the look and feel, and the brand strategy for the cloud storage service.

Logo

I went with a cloud-memory theme (the cloud is for cloud storage). I also came up with the brand name “Rem”, which is reminiscent of “memory” words.

Color Palette

This color scheme is dominated by the blue colors, which give an impression of calmness, reliability, and integrity, while yellow provides a warm counterpoint and gives a feeling of joy. I wanted the colors to feel more organic, so I added tones to the pure hues.

Although the primary focus for the minimum viable product for our cloud storage service is a web browser based application, I decided to create the mobile mockup first to ensure scalability.

The desktop mockup of the dashboard was then created using a combination of the layout used in the wireframe and the visual style used in the mobile mockup. I also made a series of adjustments to elements of the dashboard page to solve issues encountered during the wireframe testing phase.

Preference Testing

Before I moved on to usability testing for the high fidelity prototypes, I turned to my users to help me make some design choices. Preference tests are a great tool to use to help me decide between two design choices about a specific element.

70%
30%

Test One: Overlay Color

The blue toned background was preferred 70% of the time. Users who preferred this option cited that the blue tone actually provides better contrast between the darkened area and the brighter (interactable) areas. Users also found this choice to be more visually appealing.

Test Two: Mobile Dashboard

Out of 14 participants, there was an exactly 50/50 split between the two screens. I decided to use the dashboard on the right because it’s more important for the user to understand the content at a glance. The dashboard on the left may have more visual appeal, but may cause greater frustration in usage.

User Testing

User Testing - Round Two

In this stage of usability testing, I used the same general script outline and task list as I used for the first usability testing phase, but expanded the script to include the mobile prototype. Using the same format for the second usability testing phase will help us to compare the performance of the first prototype to the second prototype, and evaluate whether our solutions were effective. The goals of this usability testing phase are:

  1. Determine whether I successfully solved issues identified in wireframe usability test.

  2. Identify areas where the testers struggle to understand or use the high fidelity mockup (ie. issues with usability).

  3. Receive feedback on the look and feel of the mockup (ie. issues with visual design).

During this testing phase, I tested the desktop and mobile prototypes with three users, each of whom provided me with 25-35 issues to be considered for further improvement. I identified the top 3-4 issues encountered by each user to prioritize for improvement, which included:

Problem

Confused that the “gallery” opened inside of the app - expected to open the native gallery app (mobile)

Solution

Change the page title to “Upload” and move the folder name above the gallery image tiles

Problem

Can’t distinguish what is part of the “menu” vs the folder structure in the text editor navigation

Solution

Use a tile-based system to organize the folder content. Base the visuals on the mobile dashboard

Problem

Dashboard seems too busy because everything is different sizes - seems disorganized

Solution

Change the size and spacing of the content tiles and change tile icons to be less obtrusive

View the full User Testing Report

The mobile app had very few issues, but the issues with the upload process were very significant because it impacts one of the core functions of the cloud storage service: uploading to the cloud.

The last priority issue that I had to tackle was the visual hierarchy of the dashboard. One user specifically stated that the dashboard appeared to be confusing and visually cluttered, and the other two users took a few seconds to make sense of the page, which indicates to me that the dashboard is poorly organized and does not accomplish our original goal of creating a strong organizational system.

In the updated version of the dashboard, the new layout is a single column with one folder per row with its contents displayed horizontally across the row. I then made other visual changes to reduce the clutter.
I then took the visual changes that I made on the dashboard, and recreated those changes across various pages and elements on the desktop mockup and mobile mockup in order to create a consistent visual experience.

Branding Update

After completing the visual changes, there was one last issue that I wanted to tackle. All three of the users were initially confused about what kind of service the website was offering. The source of the confusion was the name of the brand itself. To address this issue, I changed the name of the brand to “Remkeep”, which gives more of an impression of storage, and helps dissociate the brand from the idea of sleeping.

The completed prototypes accomplish our original objective of allowing users to seamlessly organize personal files and web content together in a visually oriented space. By solving issues encountered during several stages of user testing, I was able to create a website that was intuitive for users to understand and navigate.

What I Learned

Designing Remkeep helped me to see what the design process looks like, from the first seeds of conception to a polished visual design. I experienced the impact the initial ideation, planning and research had on every stage leading up to the final delivery. I also learned that users are our most valuable resource. Our users guided me through the entire creation of Remkeep, from the surveys to the usability tests, providing me with valuable information to create an attractive and intuitive product.
